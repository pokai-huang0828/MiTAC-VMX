## Part 2 · 客戶最常問的 5 個決策

### Q1.「我用哪個 portal?」
| 角色 | 答案 |
|------|------|
| 經銷商 / 神達內部 | Master Portal(看到所有客戶的設備池) |
| 車隊老闆 / 司機主管 | Fleet Portal(只看到自己的車隊) |
| 同時要管多個車隊的 IT | 兩個都要(Master 派發、Fleet 運營) |

### Q2.「我能看到什麼?」
- **行程(Trip)**:每台車每天的開車紀錄(時間、里程、軌跡、駕駛是誰)
- **事件(Event)**:5 種類型 — 急煞、跟車過近、超速、ADAS 校正、DMS 校正
- **影片**:事件當下前向 + 駕駛艙雙鏡頭片段
- **報表**:駕駛安全評分、車隊每日活動量、設備健康度

### Q3.「資料能匯出嗎?保留多久?」
- **Trip / Event 清單**:可匯出(CSV),日期範圍可選
- **影片**:保留 **180 天**,過期會自動刪(每筆事件頁顯示倒數)
- 想延長保留期 → 找你的服務窗口談

### Q4.「VisionMax 有 4 種訂閱方案,我該選哪個?」

| 方案 | 包含什麼 | 適合誰 |
|------|---------|--------|
| **Standard** | 急煞 / 急轉 / 撞擊偵測 + GPS + 行車影片 | 只要基本行為紀錄 + 事故佐證 |
| **Advanced** | Standard 全部 + **車外輔助**(前車碰撞警告、車道偏離、跟車過近) | 想做駕駛訓練 + 防車外事故 |
| **Pro** | Advanced 全部 + **駕駛艙監控**(分心、打電話、滑手機、疲勞、安全帶) | 要全方位駕駛行為管理 |
| **Suspend** | 暫停服務(不收費,設備保留)| 季節性不開的車輛 / 暫停車輛 |

> 詳細定價請洽你的業務窗口

### Q5.「設備出問題我從哪裡看?」
- **經銷商**:Master Portal → Diagnostics(看跨車隊哪幾台有問題)
- **車隊客戶**:Fleet Portal → Devices(看自己車隊各台 ADAS / DMS 健康狀態)

## Part 3 · Master Portal 7 頁速覽

### 1. Dashboard
➤ **用途**:你管轄所有設備的即時總覽
➤ **能做的決定**:看到哪幾類問題在發生 → 分派人去處理
➤ **看到什麼**:設備總數、已派發 vs 未派發比例、健康 vs 異常、5 種異常分類

### 2. Analysis
➤ **用途**:期間設備分配 + 訂閱方案演變圖
➤ **能做的決定**:看哪些方案賣得好 / 哪些車隊在升降級
➤ **看到什麼**:過去一週(可調)的方案數量折線圖

### 3. Management > Inventory
➤ **用途**:設備池(待分配 vs 已分配)
➤ **能做的決定**:把新進貨設備派給客戶,或把退回設備重派
➤ **看到什麼**:設備清單 + IMEI / 機型 / 召回紀錄 / 連線時間

### 4. Management > Diagnostics
➤ **用途**:跨車隊設備健康檢查
➤ **能做的決定**:鎖定異常設備,通知對應客戶或派 FAE
➤ **看到什麼**:✓ 沒問題 / ⚠ 有問題 兩個數字 + 異常設備清單

### 5. Management > Fleets
➤ **用途**:你管轄的所有車隊清單
➤ **能做的決定**:新建車隊 / 加帳號 / 調整車隊資訊
➤ **看到什麼**:車隊名 / 帳號數 / 設備數 / 創建日期

### 6. Management > User Account
➤ **用途**:管理可登入 Master Portal 的人(經銷商員工)
➤ **能做的決定**:加新員工 / 停用離職員工 / 調權限(Admin / Viewer)
➤ **看到什麼**:用戶名 / Email / 角色 / 啟用狀態

### 7. User Activity Logs
➤ **用途**:Master 層級登入登出稽核
➤ **能做的決定**:查可疑登入 / 跨組織存取
➤ **看到什麼**:時間 / 用戶 / Email / 動作 / IP

## Part 4 · Fleet Portal 19 頁速覽(按使用情境分組)

### 🚗 每天看的(2 頁)

#### Map Overview
➤ **用途**:即時車隊位置地圖
➤ **能做的決定**:看哪些車正在路上 / 偏離預定路線
➤ **看到什麼**:世界地圖 + 設備 pin + 左側設備清單(可搜尋)

#### Dashboard
➤ **用途**:車隊一日總覽
➤ **能做的決定**:當日事件多不多?有沒有要立刻關注的駕駛?
➤ **看到什麼**:Live Trips / Finished Trips / Unknown Drivers 數字 + 事件趨勢 + 駕駛安全評分排名

### 🛣️ Trip 管理(3 頁)

#### Trips > Trip List
➤ **用途**:所有行程清單(預設過去 30 天)
➤ **能做的決定**:篩出特定車或駕駛的行程查證
➤ **看到什麼**:每筆 trip 的時間 / 里程 / 駕駛 / 設備

#### Trip Detail
➤ **用途**:單筆行程的完整 timeline
➤ **能做的決定**:查這趟有沒有違規 / 異常事件
➤ **看到什麼**:GPS 軌跡地圖 + 事件時間軸(Start / DMS / ADAS / 違規 / End)+ 調閱影片按鈕

#### Trips > Trip Report
➤ **用途**:期間行程量化報表
➤ **能做的決定**:衡量車隊整體活動量是否正常
➤ **看到什麼**:Trip 數 / 總里程 / 總時長 + 趨勢圖 + 駕駛排名

### 🚨 事件 / 客訴管理(5 頁)

#### Safety > Events
➤ **用途**:當日所有事件卡片牆
➤ **能做的決定**:快速掃過今天有什麼狀況
➤ **看到什麼**:事件按類型分組(急煞 / 超速 / 跟車過近 等)+ 影片縮圖 + 影片保留剩餘天數

#### Event Detail(單筆事件)
➤ **用途**:單一事件深入分析
➤ **能做的決定**:這事件要不要記過?要不要做駕駛訓練?要不要丟棄?
➤ **看到什麼**:雙鏡頭影片 + 事件位置地圖 + 速度時間圖 + 加速度時間圖

#### Safety > File Retrieval
➤ **用途**:管理已調閱的影片檔案
➤ **能做的決定**:下載 / 整理事故證據
➤ **看到什麼**:已調閱影片清單

#### Safety > Safety Reports
➤ **用途**:車隊安全月報 / 期間報表
➤ **能做的決定**:看趨勢有沒有改善,跟主管 / 保險彙報
➤ **看到什麼**:駕駛 / 車輛 / 事件分類 排名與趨勢

#### Safety > Coaching
➤ **用途**:駕駛訓練教材庫
➤ **能做的決定**:挑選有教育意義的事件,跟司機坐下來檢討
➤ **看到什麼**:被標記為「教材」的事件清單(從 Event Detail 標記過來)

### 🔧 設備 / 駕駛 / 車輛 / 區域(4 頁)

#### Management > Devices
➤ **用途**:車隊內所有 K-series 設備清單
➤ **能做的決定**:看哪台設備需要 FAE 介入(ADAS / DMS Health 顯示)
➤ **看到什麼**:設備 ID / 機型 / 上線狀態 / 韌體版本 / AI 健康度

#### Management > Drivers
➤ **用途**:車隊駕駛員清單(綁 RFID 卡)
➤ **能做的決定**:加新司機 / 停用離職司機 / 看哪位司機開哪些車
➤ **看到什麼**:司機名 / 員工編號 / IC 卡序號 / Email

#### Management > Vehicles
➤ **用途**:車輛資料(綁設備)
➤ **能做的決定**:確認車牌 / 車型 / 設備配對
➤ **看到什麼**:VIN / 車型 / 對應的設備 ID

#### Management > Geofences
➤ **用途**:地理圍欄設定
➤ **能做的決定**:設定哪些區域進入 / 離開要警示
➤ **看到什麼**:已建圍欄清單 + 地圖

### ⚙ 設定(1 頁)

#### Configurations
➤ **用途**:車隊全域設備行為調整
➤ **能做的決定**:依車型(轎車 / 中型 / 重型)調觸發門檻、音量、語音、超速安全分數
➤ **看到什麼**:5 個設定 tab — Device Settings / Sensor Event / AI Event / Road Safety / Safety Score

### 👥 行政管理(4 頁)

#### Administration > Accounts & Permissions
➤ **用途**:Fleet Portal 帳號管理
➤ **能做的決定**:加管理員 / 設角色權限
➤ **看到什麼**:帳號清單 + 角色 + 啟用狀態

#### Administration > Fleets
➤ **用途**:車隊本身的資料(logo / 時區 / 地址 / 電話)+ Contract Fleets 設定
➤ **能做的決定**:更改車隊基本資料
➤ **看到什麼**:Fleet 資訊表單

#### Administration > Device Usage
➤ **用途**:設備使用率
➤ **能做的決定**:看哪些設備閒置該回收 / 重派
➤ **看到什麼**:每台設備使用紀錄

#### Administration > User Activity Logs
➤ **用途**:Fleet 內帳號操作稽核
➤ **能做的決定**:查可疑操作 / 違規授權
➤ **看到什麼**:操作時間 / 用戶 / 動作

## Part 5 · 三大 Presentation Demo 流程(各 5 分鐘)

### 流程 A · 給「車隊老闆」看(關心 ROI + 風險)

> 30 秒對焦:你最在意「車隊有沒有出事」「司機表現好不好」「保險有沒有依據」

```
1. Dashboard               (15 秒)→ 你的車隊一目了然
2. Trip Report             (60 秒)→ 本月跑了多少 / 總里程 / 趨勢
3. Safety Reports          (90 秒)→ 哪幾個司機需要訓練 / 風險最高的車
4. Event Detail (隨選一筆)  (90 秒)→ 點進去看雙鏡頭影片 + 速度圖,證明「這真的有發生」
5. Coaching               (45 秒)→ 把這事件加進訓練教材,跟司機溝通
```

**收尾問題**:「你想先看看哪個司機 / 哪個事件?」

### 流程 B · 給「IT 經理」看(關心可控性 + 帳號)

> 30 秒對焦:你關心「設備健康」「能不能管權限」「資料能不能匯出 / 對接系統」

```
1. Devices                 (60 秒)→ 看到所有車的設備健康度 + 韌體版本
2. Configurations          (90 秒)→ 改車型門檻、音量、語音 → 全 fleet 套用
3. Accounts & Permissions  (60 秒)→ 加管理員、設權限,SSO 等可談
4. User Activity Logs      (45 秒)→ 稽核紀錄,合規好交差
5. Geofences               (45 秒)→ 圍欄,可以對接你們現有 ERP
```

**收尾問題**:「你的內部系統有什麼想對接的?(API / Webhook / Custom SDK)」

### 流程 C · 給「司機主管」看(關心人 + 行為)

> 30 秒對焦:你關心「司機怎麼開」「事故有沒有影片佐證」「能不能拿來教育」

```
1. Map Overview            (30 秒)→ 即時看到每台車在哪
2. Trip List → 點某筆       (90 秒)→ 看單一行程細節 + 軌跡 + 事件 timeline
3. Events 卡片牆           (60 秒)→ 今天 / 本週發生的事件分類
4. Event Detail            (90 秒)→ 點進去看 7 種駕駛艙偵測標籤(打電話 / 滑手機 / 安全帶 / 抽菸 等)
5. Coaching                (60 秒)→ 把事件變教材,做訓練紀錄
```

**收尾問題**:「你目前怎麼跟司機檢討事故?(可以省下你這部分時間)」
