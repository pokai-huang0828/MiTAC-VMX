📺 例子 1:轉頭 — 分心 or 看後照鏡?

場景

商用車駕駛在行駛中突然轉頭。

舊 Server AI 判讀

  • 規格:轉頭角度 > 閾值 = 分心
  • 結果:❌ 誤報 → 管理者收到警報但實際駕駛安全

新 VLM Server AI 判讀

  • 連續影片分析:駕駛轉頭方向、視線停留時間、頸部動作模式
  • 推論:駕駛在「看後照鏡」(正常駕駛行為)
  • 結果:✅ 正確不警報 → 顯著降低假警報、管理者信任度提升

📱 例子 2:用手機 — 鏡頭看不到怎麼判?

場景

駕駛把手機放在大腿或方向盤下方,鏡頭看不到手機本身。

舊 Server AI 判讀

  • 規格:鏡頭未拍到手機 = 沒用手機
  • 結果:❌ 漏判 → 駕駛實際在用手機,系統卻沒抓到

新 VLM Server AI 判讀

  • 連續影片推論:駕駛視線持續下移、手部低位動作、頭部姿態
  • 推論:駕駛正在使用手機(即使看不到手機本體)
  • 結果:✅ 正確判定 + 警報 → 提升危險駕駛覆蓋率

1. 研發策略:從 Server AI 進化到 VLM 視覺語言模型架構

MiTAC 正在引領車聯網 AI 演算法研發,推出首個結合行車記錄器邊緣與雲端 VLM(Vision Language Model)視覺語言模型的軟硬體混合架構。透過自研演算法與雲端二次分析,讓邊緣車載設備在地端獲得過去僅限於資料中心等級的 AI 視覺辨識效能。

2026 年我們進一步將平台從 Server AI 升級到 VLM 多模態架構,讓系統不僅能「偵測事件」更能「描述事件」 — 自動生成自然語言的事件摘要與駕駛行為分析報告,顯著降低車隊管理者人工判讀成本,並對齊 GenAI 浪潮下對「可解釋 AI」的評審期待。

2. 核心研發布局:5 大新功能 + 完整 MLOps

2026 年 MiTAC Server AI 平台導入 5 項新世代功能模組,皆已通過內部 production 驗證:

  • 車道偏離警示(LDWS):結合道路風險上下文感知,降低高速公路單調路段意外
  • 駕駛眼動穩定度演算法(Eye Stable Rate):解決駕駛戴墨鏡導致眼動偵測失準的業界痛點
  • 攝影機自動校正(Camera Auto-Calibration):讓安裝位置誤差不影響辨識精度
  • 駕駛行為信心評分(Behavior Confidence Scoring):降低 DMS 假警報
  • 駕駛行為比對模型(Behavior Comparative Model):跨車隊行為基準分析

研發團隊建立完整 MLOps 模型治理框架,涵蓋訓練資料管理、模型版本控制、生產環境 A/B testing 與快速回滾機制,確保跨地區、跨客戶的模型品質一致性,符合 2026 年精品獎評審對 AI 治理新增的審核重點。

3. 智慧財產:Taiwan 自研 5 項 AI 專利

法務團隊於產品開發階段即參與審查,確保專利性(新穎性、進步性、實用性)與商業價值(市場潛力、實施難易度、經濟效益)雙軌評估。MiTAC 累積 5 項影像辨識與車隊管理領域的 Taiwan 自研 AI 專利:

  • I708696:停車行為監測方法(UBI 保險應用)
  • I745752:特殊交通標誌辨識技術
  • I778774:斜坡停車輔助辨識
  • 202432411:事故與駕駛異常行為分析系統
  • I768989:遠端資料更新機制

2026 年正進一步申請 VLM 多模態車載演算法、Camera Auto-Calibration 與 Edge-Cloud Sync 等新世代 AI 專利,持續累積本土智財。

4. 全球 R&D 協作:Taiwan 研發 + 國際落地

研發中心位於 Taiwan,與歐、美、澳客戶端 deployment team 透過 Master/Fleet 雙 Portal 架構建立即時技術回饋通路。2025-2026 年成功解決跨 3 家國際車隊客戶於不同國家同時遭遇邊緣 AI 誤報的事件 — 透過系統性根因分析,3 週內同步推送 fix 至全球客戶,展現 Taiwan 研發中心對全球 production 環境的快速應變能力。

1. Server AI 雙層架構:邊緣捕捉 + 雲端 VLM 推論

MiTAC Server AI 雲端智慧分析平台採「邊緣事件觸發 + 雲端 VLM 二次分析」雙層架構。邊緣行車記錄器負責即時影像捕捉與基本判讀,觸發事件後將關鍵影像上傳雲端伺服器,由 VLM 視覺語言模型進行高精度二次分析。此設計徹底突破車載晶片算力受限的業界瓶頸,讓客戶不需替換硬體即可享有資料中心等級 AI 效能。

2. Design Rationale:為解決真實痛點而生的功能

不同於市場上以「功能列表」思維設計的車隊系統,Server AI 平台每項功能皆對應一個業界真實痛點:

  • 為解決商用車駕駛長期戴墨鏡導致眼動偵測失準 → 設計 Eye Stable Rate 動態閾值演算法
  • 為解決車隊安裝位置不一致導致辨識誤差 → 設計 Camera Auto-Calibration
  • 為解決 DMS 誤報率偏高造成警報疲勞 → 設計行為信心評分 + 比對模型
  • 為解決事故釐清需大量人工調閱影像 → 設計 VLM 自動生成事件摘要

3. Privacy by Design:符合 GDPR / 歐盟 AI Act

平台於雲端深度學習運算時,同步內建影像遮蔽與去識別技術。針對車外行人臉部、車牌號碼等敏感資訊進行自動馬賽克處理,符合 EU AI Act(2024)、GDPR、加州 CCPA 與多國個資法規範。這不只是「合規功能」,而是 Privacy-by-Design 的架構選擇 — 從資料收集源頭即進行隱私保護,而非事後補丁。

4. 6 重國際認證 + 雙 Portal UX 分層設計

平台通過 ISO 9001、ISO 14001、QC 080000、CE、FCC、WEEE 共 6 重國際標準。並以雙 Portal 設計分層使用者體驗:

  • Master Portal:提供總管理者跨車隊鳥瞰視角,即時掌握全球部署狀態
  • Fleet Portal:提供操作員單一車隊深度操作介面,降低日常使用認知負擔

此分權治理架構符合大型企業車隊組織內部分層管理需求。

5. 雲端整合 + 低功耗:邊緣輕量化設計

邊緣裝置採超低功耗車載晶片設計,搭配 Server AI 後端集中運算,終端裝置得以大幅輕量化,有效延長設備壽命並降低能源消耗。配合 OTA(Over-the-Air)遠端更新與雲端即時診斷,實現零中斷運作與低能耗效能平衡。

6. 模組化架構 + 多模組整合彈性

整合 G-Sensor、GPS、LTE、Bluetooth、CANBUS、NFC 等多模組,以彈性 API 對接既有車聯網平台。無論是大型物流、巴士、工程車、保全車隊或保險業客戶,皆可依需求快速整合,展現穩定可靠的運行品質與智慧連結效能。

📄 3.品質.docx — Kenny 重寫版 5/15 校正

1. 從規格判定到意圖判定:AI 模型品質的下一個層次

不同於市場上單純依據規格參數(轉頭角度、視線方向)判定駕駛行為的車載 AI,神達 Server AI 平台導入視覺語言模型(Visual Language Model),進階具備「意圖判定」能力。

例如駕駛轉頭時,系統能透過連續影像理解駕駛是「看後照鏡」(正常駕駛行為)而非分心,避免假警報;駕駛使用手機時,即使鏡頭未直接拍到手機本體,系統仍能從視線下移、手部動作推論駕駛意圖。這項進階品質提升 AI 在實際 production 環境的可信度,將車載 AI 從「偵測」升級到「理解」。

2. 創新品質符合車隊客戶潛在需求

Server AI 平台整合 7-channel 360 度全方位監控、AI 駕駛行為分析、4G 雲端聯網,以及 VisionMax 後台管理。透過完善後台,車隊管理者可即時監控、預警、管控駕駛風險。針對保險業應用,平台可即時偵測事故、上傳影片、重建事件,並支援雙向通話讓保險服務人員介入,加速理賠流程。硬體強化「易安裝、難拆卸」特性,確保駕駛員無法任意拆卸設備或修改參數。

3. 設計、製造過程嚴格控制品質

MiTAC 通過 ISO 9001 / ISO 14001 / QC 080000 認證,採用 V-Model 軟硬體開發流程,嚴格管控 EVT / DVT / PVT / MP 各階段品質驗證,每階段由品質單位驗證產品功能、耐用性、可靠性。軟體服務方面,專業測試單位逐階段把關,並執行伺服器與 App 弱點掃描與滲透測試,維護資安。
